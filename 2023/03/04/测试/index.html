Win10如何搭建自己的hexo博客

以下内容出现的括号表示内容需自行填写

准备

从node.js官网中下载node.js，下稳定版，也就是左边的版本。
下载好之后按照提示一路next直到安装完成就好，但选择存储地址时默认为c盘，记得修改。

部署hexo

这一步真的绕了很多弯路。

打开命令行窗口，按以下指令输入

可以在搜索框内按cmd快速打开。
注意，全部指令都要用英文键盘输入
注意，全部指令都要用英文键盘输入
注意，全部指令都要用英文键盘输入
node -v node版本
npm -v npm版本
npm install -g cnmp --registry=https://registry.npm.taobao.org 配置淘宝镜像源
cnpm install -g hexo-cli 安装hexo工具
依次输入以上指令之后，查看版本安装成功与否：hexo -v
一般来说前面几步不会出现任何问题。接下来开始配置blog。
首先选择一个你喜欢的盘，创建一个名为blog的空文档，比如说你创建在D盘，那么按以下指令进行操作。
d:
cd blog
进行到这一步时，需要注意一个点，我在这卡了很久，还跑去傻乎乎地安装sshd服务，最后发现根本不需要。
在盘中找到blog文件的位置，打开其属性，找到“安全”那个模块并点击，点击里面的“编辑”，把其权限全部打开，不然在初始化hexo时会出现报错。
完成这一步之后，按以下指令进行操作。
hexo init 初始化hexo
hexo s 启动hexo
到这一步我们会得到一个网址，如图

在浏览器中输入如上地址，就可以进入hexo页面，那就是你的博客。

部署

这里我是选择部署到GitHub上，其他的如腾讯云之类的也可以，看自己需要。
以下关于GitHub的操作都不再赘述，如有不懂请自行百度
首先我们需要拥有一个GitHub账号，随后在自己的账号上新建一个仓库。
仓库命名为：（你的GitHub账户名）.github.io
新建仓库后，我们可以得到这个仓库的链接，将该链接保存下来。
接着在命令行中执行以下指令：
cnpm install --save hexo-deployer-git
随后我们打开blog文件，会发现里面有一个名为**_config.yml**的文件。
打开该文件，翻到代码最末，可以看到如图

按如下填写，注意每个：后都有空格：
type: git
repo: (你刚刚复制的仓库地址)
branch: master
填写完成后保存并推出。
最后再执行如下指令：
hexo d
到此，已成功将hexo部署在GitHub中。之后再打开自己的hexo博客不需要再用之前所用的链接，而是可以直接用仓库名字登录，也就是上述所说的

仓库命名为：（你的GitHub账户名）.github.io
将其复制粘贴到浏览器中就可以直接打开自己的hexo博客。

此时部署就已经全部完成了。

如果需要重新配置_config.yml文件，按如下指令操作：
hexo clean
hexo g
hexo s
hexo d

小提示：每次要在命令行窗口中进入hexo都要先进入blog文件

如何发布博客

这里提供一下我自己用的方法，因为我自己在这里也折腾了一段时间。
首先依旧是打开命令行窗口。
接下来依旧以blog文件在D盘举例：
d:
cd blog
hexo clean
这是进入hexo并清楚缓存。随后按如下指令新建博客：
hexo n "(博客title)"
输入之后，我们可以指令中提示的位置找到我们新建的文档，如图中的created所示

打开文档之后，我们可以直接在文档内进行编剧，然后保存。
按如下步骤将保存好的博客同步到hexo内：
在命令内输入：
hexo g 部署文章
hexo s 运行本地站点，查看运行结果
hexo g -d 发布
之后刷新博客页面，就可以看到新增的博客了,可能会存在一定延迟。
